Discord through AgentRoot. Your agent never holds the Discord API key — every call routes through the proxy, the key is injected from the vault, and the call is classified, gated, and audited.

## Setup

1. Dashboard → **Connect** → Discord
2. Paste your Discord API key into the secure iframe
3. Bind the key to one or more agents
4. Point your code at `https://proxy.agentroot.app/discord`

For the conventions every example assumes — required headers, error codes, idempotency — see [Conventions](../conventions).

## Action class

Discord endpoints classify as **Communication** by default. See [Action Classes](/action-classes) for what governs this class and the relevant per-class auto-kill thresholds. To override per-route, see [Action Class Mapping](../action-class-mapping#overriding-classifications).

## Calling Discord through AgentRoot

Representative call — `POST /api/v10/channels/{channel_id}/messages`:

<CodeGroup>
```bash cURL
curl -X POST 'https://proxy.agentroot.app/discord/api/v10/channels/{channel_id}/messages' \
      -H "Authorization: Bearer $AGENTROOT_TOKEN" \
      -H "X-AgentRoot-ID: $AGENTROOT_AGENT_ID" \
      -H "Content-Type: application/json" \
      -d '{
  "content": "hello from AgentRoot"
}'
```

```typescript TypeScript
await ar.proxy({
      provider: 'discord',
      path: '/api/v10/channels/{channel_id}/messages',
      method: 'POST',,
      body: {
  "content": "hello from AgentRoot"
},
    });
```

```python Python
import os, requests
requests.post(
      "https://proxy.agentroot.app/discord/api/v10/channels/{channel_id}/messages",
      headers={
        "Authorization": f"Bearer {os.environ['AGENTROOT_TOKEN']}",
        "X-AgentRoot-ID": os.environ["AGENTROOT_AGENT_ID"],
      },
      json={
  "content": "hello from AgentRoot"
},
    )
```
</CodeGroup>
