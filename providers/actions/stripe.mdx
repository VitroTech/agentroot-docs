Stripe through AgentRoot. Your agent never holds the Stripe API key — every call routes through the proxy, the key is injected from the vault, and the call is classified, gated, and audited.

## Setup

1. Dashboard → **Connect** → Stripe
2. Paste your Stripe API key into the secure iframe
3. Bind the key to one or more agents
4. Point your code at `https://proxy.agentroot.app/stripe`

For the conventions every example assumes — required headers, error codes, idempotency — see [Conventions](../conventions).

## Action class

Stripe endpoints classify as **Transactions** by default. See [Action Classes](/action-classes) for what governs this class and the relevant per-class auto-kill thresholds. To override per-route, see [Action Class Mapping](../action-class-mapping#overriding-classifications).

## Calling Stripe through AgentRoot

Representative call — `POST /v1/refunds`:

<CodeGroup>
```bash cURL
curl -X POST 'https://proxy.agentroot.app/stripe/v1/refunds' \
      -H "Authorization: Bearer $AGENTROOT_TOKEN" \
      -H "X-AgentRoot-ID: $AGENTROOT_AGENT_ID" \
      -H "Content-Type: application/json" \
      -d '{
  "charge": "ch_..."
}'
```

```typescript TypeScript
await ar.proxy({
      provider: 'stripe',
      path: '/v1/refunds',
      method: 'POST',,
      body: {
  "charge": "ch_..."
},
    });
```

```python Python
import os, requests
requests.post(
      "https://proxy.agentroot.app/stripe/v1/refunds",
      headers={
        "Authorization": f"Bearer {os.environ['AGENTROOT_TOKEN']}",
        "X-AgentRoot-ID": os.environ["AGENTROOT_AGENT_ID"],
      },
      json={
  "charge": "ch_..."
},
    )
```
</CodeGroup>
