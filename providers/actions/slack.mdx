Slack through AgentRoot. Your agent never holds the Slack API key — every call routes through the proxy, the key is injected from the vault, and the call is classified, gated, and audited.

## Setup

1. Dashboard → **Connect** → Slack
2. Paste your Slack API key into the secure iframe
3. Bind the key to one or more agents
4. Point your code at `https://proxy.agentroot.app/slack`

For the conventions every example assumes — required headers, error codes, idempotency — see [Conventions](../conventions).

## Action class

Slack endpoints classify as **Communication** by default. See [Action Classes](/action-classes) for what governs this class and the relevant per-class auto-kill thresholds. To override per-route, see [Action Class Mapping](../action-class-mapping#overriding-classifications).

## Calling Slack through AgentRoot

Representative call — `POST /api/chat.postMessage`:

<CodeGroup>
```bash cURL
curl -X POST 'https://proxy.agentroot.app/slack/api/chat.postMessage' \
      -H "Authorization: Bearer $AGENTROOT_TOKEN" \
      -H "X-AgentRoot-ID: $AGENTROOT_AGENT_ID" \
      -H "Content-Type: application/json" \
      -d '{
  "channel": "#general",
  "text": "hello from AgentRoot"
}'
```

```typescript TypeScript
await ar.proxy({
      provider: 'slack',
      path: '/api/chat.postMessage',
      method: 'POST',,
      body: {
  "channel": "#general",
  "text": "hello from AgentRoot"
},
    });
```

```python Python
import os, requests
requests.post(
      "https://proxy.agentroot.app/slack/api/chat.postMessage",
      headers={
        "Authorization": f"Bearer {os.environ['AGENTROOT_TOKEN']}",
        "X-AgentRoot-ID": os.environ["AGENTROOT_AGENT_ID"],
      },
      json={
  "channel": "#general",
  "text": "hello from AgentRoot"
},
    )
```
</CodeGroup>
