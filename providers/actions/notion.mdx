Notion through AgentRoot. Your agent never holds the Notion API key — every call routes through the proxy, the key is injected from the vault, and the call is classified, gated, and audited.

## Setup

1. Dashboard → **Connect** → Notion
2. Paste your Notion API key into the secure iframe
3. Bind the key to one or more agents
4. Point your code at `https://proxy.agentroot.app/notion`

For the conventions every example assumes — required headers, error codes, idempotency — see [Conventions](../conventions).

## Action class

Notion endpoints classify as **Writes** by default. See [Action Classes](/action-classes) for what governs this class and the relevant per-class auto-kill thresholds. To override per-route, see [Action Class Mapping](../action-class-mapping#overriding-classifications).

## Calling Notion through AgentRoot

Representative call — `POST /v1/pages`:

<CodeGroup>
```bash cURL
curl -X POST 'https://proxy.agentroot.app/notion/v1/pages' \
      -H "Authorization: Bearer $AGENTROOT_TOKEN" \
      -H "X-AgentRoot-ID: $AGENTROOT_AGENT_ID" \
      -H "Content-Type: application/json" \
      -d '{
  "parent": {
    "database_id": "<id>"
  },
  "properties": {
    "Name": {
      "title": [
        {
          "text": {
            "content": "New page"
          }
        }
      ]
    }
  }
}'
```

```typescript TypeScript
await fetch('https://proxy.agentroot.app/notion/v1/pages', {

    "Name": {
      "title": [
        {
          "text": {

          }
        }
      ]
    }
  }
}),
});
```

```python Python

  }

    )
```
