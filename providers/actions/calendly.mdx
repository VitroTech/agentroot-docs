Calendly through AgentRoot. Your agent never holds the Calendly API key — every call routes through the proxy, the key is injected from the vault, and the call is classified, gated, and audited.

## Setup

1. Dashboard → **Connect** → Calendly
2. Paste your Calendly API key into the secure iframe
3. Bind the key to one or more agents
4. Point your code at `https://proxy.agentroot.app/calendly`

For the conventions every example assumes — required headers, error codes, idempotency — see [Conventions](../conventions).

## Action class

Calendly endpoints classify as **Writes** by default. See [Action Classes](/action-classes) for what governs this class and the relevant per-class auto-kill thresholds. To override per-route, see [Action Class Mapping](../action-class-mapping#overriding-classifications).

## Calling Calendly through AgentRoot

Representative call — `GET /scheduled_events`:

<CodeGroup>
```bash cURL
curl 'https://proxy.agentroot.app/calendly/scheduled_events' \
      -H "Authorization: Bearer $AGENTROOT_TOKEN" \
      -H "X-AgentRoot-ID: $AGENTROOT_AGENT_ID"
```

```typescript TypeScript
await ar.proxy({
      provider: 'calendly',
      path: '/scheduled_events',
      method: 'GET',
    });
```

```python Python
import os, requests
requests.get(
      "https://proxy.agentroot.app/calendly/scheduled_events",
      headers={
        "Authorization": f"Bearer {os.environ['AGENTROOT_TOKEN']}",
        "X-AgentRoot-ID": os.environ["AGENTROOT_AGENT_ID"],
      },
    )
```
</CodeGroup>
