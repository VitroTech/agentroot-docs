Airtable through AgentRoot. Your agent never holds the Airtable API key — every call routes through the proxy, the key is injected from the vault, and the call is classified, gated, and audited.

## Setup

1. Dashboard → **Connect** → Airtable
2. Paste your Airtable API key into the secure iframe
3. Bind the key to one or more agents
4. Point your code at `https://proxy.agentroot.app/airtable`

For the conventions every example assumes — required headers, error codes, idempotency — see [Conventions](../conventions).

## Action class

Airtable endpoints classify as **Writes** by default. See [Action Classes](/action-classes) for what governs this class and the relevant per-class auto-kill thresholds. To override per-route, see [Action Class Mapping](../action-class-mapping#overriding-classifications).

## Calling Airtable through AgentRoot

Representative call — `POST /v0/{base_id}/{table}`:

<CodeGroup>
```bash cURL
curl -X POST 'https://proxy.agentroot.app/airtable/v0/{base_id}/{table}' \
      -H "Authorization: Bearer $AGENTROOT_TOKEN" \
      -H "X-AgentRoot-ID: $AGENTROOT_AGENT_ID" \
      -H "Content-Type: application/json" \
      -d '{
  "fields": {
    "Name": "From the agent"
  }
}'
```

```typescript TypeScript
await ar.proxy({
      provider: 'airtable',
      path: '/v0/{base_id}/{table}',
      method: 'POST',,
      body: {
  "fields": {
    "Name": "From the agent"
  }
},
    });
```

```python Python
import os, requests
requests.post(
      "https://proxy.agentroot.app/airtable/v0/{base_id}/{table}",
      headers={
        "Authorization": f"Bearer {os.environ['AGENTROOT_TOKEN']}",
        "X-AgentRoot-ID": os.environ["AGENTROOT_AGENT_ID"],
      },
      json={
  "fields": {
    "Name": "From the agent"
  }
},
    )
```
</CodeGroup>
