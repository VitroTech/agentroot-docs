Asana through AgentRoot. Your agent never holds the Asana API key — every call routes through the proxy, the key is injected from the vault, and the call is classified, gated, and audited.

## Setup

1. Dashboard → **Connect** → Asana
2. Paste your Asana API key into the secure iframe
3. Bind the key to one or more agents
4. Point your code at `https://proxy.agentroot.app/asana`

For the conventions every example assumes — required headers, error codes, idempotency — see [Conventions](../conventions).

## Action class

Asana endpoints classify as **Writes** by default. See [Action Classes](/action-classes) for what governs this class and the relevant per-class auto-kill thresholds. To override per-route, see [Action Class Mapping](../action-class-mapping#overriding-classifications).

## Calling Asana through AgentRoot

Representative call — `POST /api/1.0/tasks`:

<CodeGroup>
```bash cURL
curl -X POST 'https://proxy.agentroot.app/asana/api/1.0/tasks' \
      -H "Authorization: Bearer $AGENTROOT_TOKEN" \
      -H "X-AgentRoot-ID: $AGENTROOT_AGENT_ID" \
      -H "Content-Type: application/json" \
      -d '{
  "data": {
    "name": "From the agent",
    "projects": [
      "<gid>"
    ]
  }
}'
```

```typescript TypeScript
await ar.proxy({
      provider: 'asana',
      path: '/api/1.0/tasks',
      method: 'POST',,
      body: {
  "data": {
    "name": "From the agent",
    "projects": [
      "<gid>"
    ]
  }
},
    });
```

```python Python
import os, requests
requests.post(
      "https://proxy.agentroot.app/asana/api/1.0/tasks",
      headers={
        "Authorization": f"Bearer {os.environ['AGENTROOT_TOKEN']}",
        "X-AgentRoot-ID": os.environ["AGENTROOT_AGENT_ID"],
      },
      json={
  "data": {
    "name": "From the agent",
    "projects": [
      "<gid>"
    ]
  }
},
    )
```
</CodeGroup>
