Firecrawl through AgentRoot. Your agent never holds the Firecrawl API key — every call routes through the proxy, the key is injected from the vault, and the call is classified, gated, and audited.

## Setup

1. Dashboard → **Connect** → Firecrawl
2. Paste your Firecrawl API key into the secure iframe
3. Bind the key to one or more agents
4. Point your code at `https://proxy.agentroot.app/firecrawl`

For the conventions every example assumes — required headers, error codes, idempotency — see [Conventions](../conventions).

## Action class

Firecrawl endpoints classify as **Writes** by default. See [Action Classes](/action-classes) for what governs this class and the relevant per-class auto-kill thresholds. To override per-route, see [Action Class Mapping](../action-class-mapping#overriding-classifications).

## Calling Firecrawl through AgentRoot

Representative call — `POST /v1/scrape`:

<CodeGroup>
```bash cURL
curl -X POST 'https://proxy.agentroot.app/firecrawl/v1/scrape' \
      -H "Authorization: Bearer $AGENTROOT_TOKEN" \
      -H "X-AgentRoot-ID: $AGENTROOT_AGENT_ID" \
      -H "Content-Type: application/json" \
      -d '{
  "url": "https://agentroot.app",
  "formats": [
    "markdown"
  ]
}'
```

```typescript TypeScript
await ar.proxy({
      provider: 'firecrawl',
      path: '/v1/scrape',
      method: 'POST',,
      body: {
  "url": "https://agentroot.app",
  "formats": [
    "markdown"
  ]
},
    });
```

```python Python
import os, requests
requests.post(
      "https://proxy.agentroot.app/firecrawl/v1/scrape",
      headers={
        "Authorization": f"Bearer {os.environ['AGENTROOT_TOKEN']}",
        "X-AgentRoot-ID": os.environ["AGENTROOT_AGENT_ID"],
      },
      json={
  "url": "https://agentroot.app",
  "formats": [
    "markdown"
  ]
},
    )
```
</CodeGroup>
