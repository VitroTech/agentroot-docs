Exa through AgentRoot. Your agent never holds the Exa API key — every call routes through the proxy, the key is injected from the vault, and the call is classified, gated, and audited.

## Setup

1. Dashboard → **Connect** → Exa
2. Paste your Exa API key into the secure iframe
3. Bind the key to one or more agents
4. Point your code at `https://proxy.agentroot.app/exa`

For the conventions every example assumes — required headers, error codes, idempotency — see [Conventions](../conventions).

## Action class

Exa endpoints classify as **Writes** by default. See [Action Classes](/action-classes) for what governs this class and the relevant per-class auto-kill thresholds. To override per-route, see [Action Class Mapping](../action-class-mapping#overriding-classifications).

## Calling Exa through AgentRoot

Representative call — `POST /search`:

<CodeGroup>
```bash cURL
curl -X POST 'https://proxy.agentroot.app/exa/search' \
      -H "Authorization: Bearer $AGENTROOT_TOKEN" \
      -H "X-AgentRoot-ID: $AGENTROOT_AGENT_ID" \
      -H "Content-Type: application/json" \
      -d '{
  "query": "agent governance frameworks",
  "numResults": 10
}'
```

```typescript TypeScript
await ar.proxy({
      provider: 'exa',
      path: '/search',
      method: 'POST',,
      body: {
  "query": "agent governance frameworks",
  "numResults": 10
},
    });
```

```python Python
import os, requests
requests.post(
      "https://proxy.agentroot.app/exa/search",
      headers={
        "Authorization": f"Bearer {os.environ['AGENTROOT_TOKEN']}",
        "X-AgentRoot-ID": os.environ["AGENTROOT_AGENT_ID"],
      },
      json={
  "query": "agent governance frameworks",
  "numResults": 10
},
    )
```
</CodeGroup>
