Browserbase through AgentRoot. Your agent never holds the Browserbase API key — every call routes through the proxy, the key is injected from the vault, and the call is classified, gated, and audited.

## Setup

1. Dashboard → **Connect** → Browserbase
2. Paste your Browserbase API key into the secure iframe
3. Bind the key to one or more agents
4. Point your code at `https://proxy.agentroot.app/browserbase`

For the conventions every example assumes — required headers, error codes, idempotency — see [Conventions](../conventions).

## Action class

Browserbase endpoints classify as **Writes** by default. See [Action Classes](/action-classes) for what governs this class and the relevant per-class auto-kill thresholds. To override per-route, see [Action Class Mapping](../action-class-mapping#overriding-classifications).

## Calling Browserbase through AgentRoot

Representative call — `POST /v1/sessions`:

<CodeGroup>
```bash cURL
curl -X POST 'https://proxy.agentroot.app/browserbase/v1/sessions' \
      -H "Authorization: Bearer $AGENTROOT_TOKEN" \
      -H "X-AgentRoot-ID: $AGENTROOT_AGENT_ID" \
      -H "Content-Type: application/json" \
      -d '{
  "projectId": "<id>"
}'
```

```typescript TypeScript
await ar.proxy({
      provider: 'browserbase',
      path: '/v1/sessions',
      method: 'POST',,
      body: {
  "projectId": "<id>"
},
    });
```

```python Python
import os, requests
requests.post(
      "https://proxy.agentroot.app/browserbase/v1/sessions",
      headers={
        "Authorization": f"Bearer {os.environ['AGENTROOT_TOKEN']}",
        "X-AgentRoot-ID": os.environ["AGENTROOT_AGENT_ID"],
      },
      json={
  "projectId": "<id>"
},
    )
```
</CodeGroup>
