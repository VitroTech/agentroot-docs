SerpAPI through AgentRoot. Your agent never holds the SerpAPI API key — every call routes through the proxy, the key is injected from the vault, and the call is classified, gated, and audited.

## Setup

1. Dashboard → **Connect** → SerpAPI
2. Paste your SerpAPI API key into the secure iframe
3. Bind the key to one or more agents
4. Point your code at `https://proxy.agentroot.app/serpapi`

For the conventions every example assumes — required headers, error codes, idempotency — see [Conventions](../conventions).

## Action class

SerpAPI endpoints classify as **Writes** by default. See [Action Classes](/action-classes) for what governs this class and the relevant per-class auto-kill thresholds. To override per-route, see [Action Class Mapping](../action-class-mapping#overriding-classifications).

## Calling SerpAPI through AgentRoot

Representative call — `GET /search.json`:

<CodeGroup>
```bash cURL
curl 'https://proxy.agentroot.app/serpapi/search.json' \
      -H "Authorization: Bearer $AGENTROOT_TOKEN" \
      -H "X-AgentRoot-ID: $AGENTROOT_AGENT_ID"
```

```typescript TypeScript
await ar.proxy({
      provider: 'serpapi',
      path: '/search.json',
      method: 'GET',
    });
```

```python Python
import os, requests
requests.get(
      "https://proxy.agentroot.app/serpapi/search.json",
      headers={
        "Authorization": f"Bearer {os.environ['AGENTROOT_TOKEN']}",
        "X-AgentRoot-ID": os.environ["AGENTROOT_AGENT_ID"],
      },
    )
```
</CodeGroup>
