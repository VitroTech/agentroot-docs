LangChain is a client-side framework. AgentRoot governs the calls LangChain makes to upstream APIs — not the LangChain library itself. The framework loads in your runtime; the AgentRoot proxy sits between your runtime and every external API the framework reaches.

## Setup

1. **Mint an agent** in the AgentRoot dashboard and grab your three env vars
2. **Bind the providers** your LangChain workflow will use (OpenAI, Slack, Stripe, whatever the agent calls)
3. **Configure LangChain** to point its LLM clients at the AgentRoot proxy — pattern depends on which underlying LLM you use

## Integration pattern

For LLM calls within LangChain, configure the underlying client (OpenAI SDK, Anthropic SDK, etc.) with AgentRoot's base URL and auth headers. See the relevant LLM provider page for the exact configuration:

- **OpenAI through AgentRoot** — [providers/llms/openai](../llms/openai) → "Drop-in client SDK integration"
- **Anthropic through AgentRoot** — [providers/llms/anthropic](../llms/anthropic)
- **Other LLM providers** — see the [LLMs catalog](../llms/openai)

For non-LLM external calls (Slack, Stripe, GitHub) that your LangChain tools make, fetch directly through the AgentRoot proxy — same two headers and base URL as every other proxied call. See [Conventions](../conventions).

## Action class governance

LangChain itself doesn't have an action class — it runs in your process. The classes apply to the calls LangChain makes to upstreams. AgentRoot's per-class auto-kill thresholds and the kill switch tiers apply to every call regardless of which framework triggered it.
