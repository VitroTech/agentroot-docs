OpenAI through AgentRoot. Your agent never holds your OpenAI API key — every call routes through the proxy, the key is injected from the vault, and the call is classified, gated, and audited.

## Setup

1. Dashboard → **Connect** → OpenAI
2. Paste your OpenAI API key (`sk-...`) into the secure iframe
3. Bind the key to one or more agents
4. Point your code at `https://proxy.agentroot.app/openai`

For the conventions every example on this page assumes — required headers, error codes, idempotency — see [Conventions](../conventions).

## Action class coverage

| Endpoint family | Class | Default posture |
|---|---|---|
| `/v1/chat/completions`, `/v1/completions`, `/v1/embeddings` | Spend | Allow with $/day cap |
| `/v1/audio/transcriptions`, `/v1/audio/translations`, `/v1/audio/speech` | Spend | Allow with $/day cap |
| `/v1/images/generations`, `/v1/images/edits`, `/v1/images/variations` | Spend | Allow with $/call cap |
| `/v1/files/*` (upload, retrieve, list) | Writes | Allow |
| `/v1/files/{id}` `DELETE` | **Destruction** | **Deny by default** |
| `/v1/fine_tuning/jobs/*` | Provisioning | HITL recommended |
| `/v1/assistants/*` create / update | Provisioning | Allow with quota |
| `/v1/threads/*` mutations | Writes | Allow |

## Endpoints

<AccordionGroup>
  <Accordion title="POST /v1/chat/completions" icon="message" description="Chat completions — the most common OpenAI call. Action class: Spend.">
    Generate a chat completion with any GPT model. Pass-through to upstream; AgentRoot adds auth and audit.

    **Action class:** Spend · **Upstream docs:** [OpenAI Chat](https://platform.openai.com/docs/api-reference/chat)

    <CodeGroup>
    ```bash cURL
    curl https://proxy.agentroot.app/openai/v1/chat/completions \
      -H "Authorization: Bearer $AGENTROOT_TOKEN" \
      -H "X-AgentRoot-ID: $AGENTROOT_AGENT_ID" \
      -H "Content-Type: application/json" \
      -d '{
        "model": "gpt-4o",
        "messages": [{"role": "user", "content": "Hello"}]
      }'
    ```

    ```typescript TypeScript
    await ar.openai('/v1/chat/completions', {
      method: 'POST',
      body: {
        model: 'gpt-4o',
        messages: [{ role: 'user', content: 'Hello' }],
      },
    });
    ```

    ```python Python
    import os, requests
    requests.post(
      "https://proxy.agentroot.app/openai/v1/chat/completions",
      headers={
        "Authorization": f"Bearer {os.environ['AGENTROOT_TOKEN']}",
        "X-AgentRoot-ID": os.environ["AGENTROOT_AGENT_ID"],
      },
      json={"model": "gpt-4o", "messages": [{"role": "user", "content": "Hello"}]},
    )
    ```
    </CodeGroup>

    Streaming is supported — set `stream: true` in the body and consume the SSE response as you would from the upstream.
  </Accordion>

## Drop-in client SDK integration

If you're using OpenAI's official client SDKs, set the base URL and pass AgentRoot's auth headers — the rest of the SDK works unchanged. The `apiKey` field is required by the SDK but unused at runtime; the real key is in the vault.

<CodeGroup>
```typescript TypeScript
import OpenAI from 'openai';

const openai = new OpenAI({
  baseURL: ar.getProxyUrl('openai') + '/v1',
  apiKey: 'unused',
  defaultHeaders: ar.getAuthHeaders(),
});

const r = await openai.chat.completions.create({
  model: 'gpt-4o',
  messages: [{ role: 'user', content: 'Hello' }],
});
```

```python Python
from openai import OpenAI

client = OpenAI(
  base_url=f"{os.environ['AGENTROOT_PROXY']}/openai/v1",
  api_key="unused",
  default_headers={
    "Authorization": f"Bearer {os.environ['AGENTROOT_TOKEN']}",
    "X-AgentRoot-ID": os.environ['AGENTROOT_AGENT_ID'],
  },
)

r = client.chat.completions.create(
  model="gpt-4o",
  messages=[{"role": "user", "content": "Hello"}],
)
```
</CodeGroup>

## Limits and notes

- **Streaming:** supported for chat completions and `audio/speech` (PCM/MP3 streaming).
- **Function/tool calling:** pass-through. AgentRoot does not interpret tool calls; classification and governance happen on the wire-level endpoint, not the semantic operation.
- **Vision (image inputs):** supported via the standard chat completions endpoint with image content blocks.
- **Realtime API (WebSocket):** not yet supported via the proxy. Use OpenAI's HTTP endpoints; WebSocket support is on the roadmap.
- **Assistants API:** supported via `/v1/threads`, `/v1/assistants`, `/v1/runs`. Tool execution within an Assistant run does *not* route through AgentRoot — only the OpenAI calls do. If you want governance on tool execution, use the regular chat completions endpoint and handle tools yourself.
