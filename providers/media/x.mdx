X (Twitter) through AgentRoot. Your agent never holds the X (Twitter) API key — every call routes through the proxy, the key is injected from the vault, and the call is classified, gated, and audited.

## Setup

1. Dashboard → **Connect** → X (Twitter)
2. Paste your X (Twitter) API key into the secure iframe
3. Bind the key to one or more agents
4. Point your code at `https://proxy.agentroot.app/x`

For the conventions every example assumes — required headers, error codes, idempotency — see [Conventions](../conventions).

## Action class

X (Twitter) endpoints classify as **Communication** by default. See [Action Classes](/action-classes) for what governs this class and the relevant per-class auto-kill thresholds. To override per-route, see [Action Class Mapping](../action-class-mapping#overriding-classifications).

## Calling X (Twitter) through AgentRoot

Representative call — `POST /2/tweets`:

<CodeGroup>
```bash cURL
curl -X POST 'https://proxy.agentroot.app/x/2/tweets' \
      -H "Authorization: Bearer $AGENTROOT_TOKEN" \
      -H "X-AgentRoot-ID: $AGENTROOT_AGENT_ID" \
      -H "Content-Type: application/json" \
      -d '{
  "text": "hello from AgentRoot"
}'
```

```typescript TypeScript
await ar.proxy({
      provider: 'x',
      path: '/2/tweets',
      method: 'POST',,
      body: {
  "text": "hello from AgentRoot"
},
    });
```

```python Python
import os, requests
requests.post(
      "https://proxy.agentroot.app/x/2/tweets",
      headers={
        "Authorization": f"Bearer {os.environ['AGENTROOT_TOKEN']}",
        "X-AgentRoot-ID": os.environ["AGENTROOT_AGENT_ID"],
      },
      json={
  "text": "hello from AgentRoot"
},
    )
```
</CodeGroup>
