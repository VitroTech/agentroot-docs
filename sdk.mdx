The AgentRoot SDK is a TypeScript library that handles proxy authentication, request routing, session token rotation, and typed error handling. Three env vars in, structured responses out. The agent never holds a key.

If you'd rather not use the SDK, every method maps to a plain HTTP call documented in the [API Reference](api-reference).

## Install

```bash
npm install @agentroot/sdk
# or: yarn add @agentroot/sdk / pnpm add @agentroot/sdk
```

## Configure

```typescript
import { AgentRoot } from '@agentroot/sdk';

const ar = new AgentRoot({
  agentId: process.env.AGENTROOT_AGENT_ID!,
  token: process.env.AGENTROOT_TOKEN!,
  proxy: process.env.AGENTROOT_PROXY!,
});
```

All three values come from the dashboard at agent registration time. Treat the token like a Bearer credential — keep it in env, never commit it. It rotates every 24 hours; the SDK handles refresh in the background.

| Option | Type | Default | Notes |
|---|---|---|---|
| `agentId` | `string` | — | EAS attestation UID |
| `token` | `string` | — | Session token (`art_live_xxx`) |
| `proxy` | `string` | — | Usually `https://proxy.agentroot.app` |
| `timeout` | `number` | `30000` | Per-request timeout in ms |
| `retries` | `number` | `2` | Retries on transient (5xx, network) failures |
| `onTokenRefresh` | `(t: string) => void` | none | Callback fired when the SDK rotates the token |

## Real example: a customer support agent

A 30-line agent that classifies an inbound message, generates a response, sends a Slack message, and (if appropriate) issues a Stripe refund. Every external call goes through AgentRoot. Every action is classified, gated, and audited.

```typescript
import { AgentRoot } from '@agentroot/sdk';

const ar = new AgentRoot({
  agentId: process.env.AGENTROOT_AGENT_ID!,
  token: process.env.AGENTROOT_TOKEN!,
  proxy: process.env.AGENTROOT_PROXY!,
});

async function handleSupportTicket(ticket: { id: string; userEmail: string; body: string; chargeId?: string }) {
  // 1. LLM classification (action class: Spend)
  const classification = await ar.openai('/v1/chat/completions', {
    method: 'POST',
    body: {
      model: 'gpt-4o-mini',
      messages: [
        { role: 'system', content: 'Classify the ticket as one of: refund_request, question, complaint.' },
        { role: 'user', content: ticket.body },
      ],
    },
  });
  const intent = classification.data.choices[0].message.content.trim();

  // 2. Generate a reply (action class: Spend)
  const reply = await ar.anthropic('/v1/messages', {
    method: 'POST',
    body: {
      model: 'claude-sonnet-4-6',
      max_tokens: 500,
      messages: [{ role: 'user', content: `Write a reply to: "${ticket.body}". Intent: ${intent}.` }],
    },
  });

  // 3. Post to Slack (action class: Communication — auto-kill if rate exceeds threshold)
  await ar.proxy({
    provider: 'slack',
    path: '/api/chat.postMessage',
    method: 'POST',
    body: {
      channel: '#support',
      text: `Ticket ${ticket.id} (${intent}): ${reply.data.content[0].text}`,
    },
  });

  // 4. Issue refund only if classified as refund_request and a charge ID exists
  //    (action class: Transactions — per-tx cap and HITL recommended)
  if (intent === 'refund_request' && ticket.chargeId) {
    await ar.proxy({
      provider: 'stripe',
      path: `/v1/refunds`,
      method: 'POST',
      body: { charge: ticket.chargeId },
    });
  }
}
```

This agent's `.env` has three values. The Slack token, OpenAI key, Anthropic key, and Stripe key are in the vault, bound to this agent in the dashboard, never visible to the runtime. If the agent goes wrong at 3am, you Disable from your phone — every Stripe refund, every Slack post, every LLM call stops in under five seconds.

## Provider shortcuts

For preset providers, use a shortcut method:

```typescript
ar.openai('/v1/chat/completions', { method: 'POST', body: { ... } });
ar.anthropic('/v1/messages', { method: 'POST', body: { ... } });
ar.google('/v1beta/models/gemini-1.5-pro:generateContent', { ... });
ar.mistral('/v1/chat/completions', { ... });
ar.groq('/openai/v1/chat/completions', { ... });
ar.cohere('/v1/chat', { ... });
```

All shortcuts are equivalent to `ar.proxy({ provider: '<name>', path, method, body })`.

## Custom providers

For providers not in the shortcut list, use `ar.proxy()` with `provider: 'custom'`:

```typescript
await ar.proxy({
  provider: 'custom',
  baseUrl: 'https://api.your-internal-service.com',
  path: '/v1/analyze',
  method: 'POST',
  body: { data: '...' },
});
```

The dashboard lets you classify custom-provider routes into action classes — by default they fall under Writes.

## Streaming

Set `stream: true` and use `for await`:

```typescript
const stream = await ar.openai('/v1/chat/completions', {
  method: 'POST',
  stream: true,
  body: {
    model: 'gpt-4o',
    messages: [{ role: 'user', content: 'Write a poem about Pinocchio.' }],
    stream: true,
  },
});

for await (const chunk of stream) {
  process.stdout.write(chunk.choices[0]?.delta?.content || '');
}
```

The proxy passes streaming responses (SSE / chunked transfer encoding) through without buffering. Latency overhead is sub-100ms at the edge.

## Session token management

Tokens have a 24-hour TTL and rotate automatically. The SDK refreshes ~10 minutes before expiry on its own.

```typescript
// Inspect token state
const status = await ar.getTokenStatus();
console.log(status.expiresAt);     // ISO timestamp
console.log(status.ttlSeconds);    // seconds remaining

// Get notified when the token rotates (e.g. to persist it)
const ar = new AgentRoot({
  ...,
  onTokenRefresh: (newToken) => {
    fs.writeFileSync('.env.agentroot', `AGENTROOT_TOKEN=${newToken}\n`);
  },
});
```

If a token is revoked from the dashboard, the next call returns `TOKEN_REVOKED` and the SDK does not retry — get a new token from the dashboard.

## Error handling

The SDK throws typed `AgentRootError` instances with stable codes. Handle the codes you care about; let the rest bubble up to your global handler.

```typescript
import { AgentRootError } from '@agentroot/sdk';

async function safeCall() {
  try {
    return await ar.openai('/v1/chat/completions', { method: 'POST', body: { ... } });
  } catch (e) {
    if (!(e instanceof AgentRootError)) throw e;

    switch (e.code) {
      case 'AGENT_KILLED':
      case 'ATTESTATION_REVOKED':
        // The operator killed this agent. Don't retry. Shut down gracefully.
        await flushBuffers();
        process.exit(0);

      case 'PROVIDER_DISABLED':
        // Tier 1 — this binding is off. Try a fallback provider.
        return await ar.anthropic('/v1/messages', { method: 'POST', body: { ... } });

      case 'RATE_LIMITED':
        // Auto-kill threshold or upstream rate limit. Honor retryAfter.
        await sleep(e.retryAfter * 1000);
        return await safeCall();

      case 'TOKEN_EXPIRED':
      case 'TOKEN_REVOKED':
        // The SDK normally rotates before this fires. If it fired, the token
        // was killed from the dashboard. Bail out.
        throw new Error('Get a new token from the dashboard');

      case 'VAULT_UNAVAILABLE':
        // Fail-closed at the proxy. AgentRoot is degraded; do not retry.
        throw new Error('AgentRoot vault unavailable — see status page');

      default:
        throw e;
    }
  }
}
```

Full code catalog:

| Code | HTTP | Meaning |
|---|---|---|
| `TOKEN_EXPIRED` | 401 | Session token TTL elapsed |
| `TOKEN_REVOKED` | 401 | Operator revoked the token |
| `TOKEN_INVALID` | 401 | Malformed or unrecognized token |
| `AGENT_KILLED` | 403 | Tier 2 — agent paused |
| `PROVIDER_DISABLED` | 403 | Tier 1 — this binding off |
| `ATTESTATION_REVOKED` | 403 | Tier 3 — onchain revocation |
| `RATE_LIMITED` | 429 | Auto-kill threshold or upstream cap |
| `UPSTREAM_ERROR` | 502 | Upstream API returned an error (forwarded) |
| `VAULT_UNAVAILABLE` | 503 | AgentRoot vault unreachable; fail-closed |

## Framework integrations

The SDK exposes helpers that let you drop AgentRoot into any framework that accepts a base URL and headers.

### LangChain

```typescript
import { ChatOpenAI } from '@langchain/openai';
import { ChatAnthropic } from '@langchain/anthropic';
import { AgentRoot } from '@agentroot/sdk';

const ar = new AgentRoot({ ... });

const llm = new ChatOpenAI({
  configuration: {
    baseURL: ar.getProxyUrl('openai') + '/v1',
    defaultHeaders: ar.getAuthHeaders(),
  },
  modelName: 'gpt-4o',
});

const claude = new ChatAnthropic({
  clientOptions: {
    baseURL: ar.getProxyUrl('anthropic'),
    defaultHeaders: ar.getAuthHeaders(),
  },
  modelName: 'claude-sonnet-4-6',
});

// Use llm and claude as you would normally — every call routes through AgentRoot.
```

### CrewAI (via OpenAI-compatible base URL)

```typescript
// CrewAI delegates to OpenAI-compatible clients. Point them at AgentRoot.
import { Crew, Agent, Task } from 'crewai';

const researcher = new Agent({
  role: 'Researcher',
  goal: 'Find current data',
  llm: {
    config: {
      baseURL: ar.getProxyUrl('openai') + '/v1',
      defaultHeaders: ar.getAuthHeaders(),
      model: 'gpt-4o',
    },
  },
});
```

### OpenAI SDK directly

```typescript
import OpenAI from 'openai';

const openai = new OpenAI({
  baseURL: ar.getProxyUrl('openai') + '/v1',
  apiKey: 'unused',                       // required by the SDK; the real key is in vault
  defaultHeaders: ar.getAuthHeaders(),
});

const response = await openai.chat.completions.create({
  model: 'gpt-4o',
  messages: [{ role: 'user', content: 'Hello' }],
});
```

### Anthropic SDK directly

```typescript
import Anthropic from '@anthropic-ai/sdk';

const claude = new Anthropic({
  baseURL: ar.getProxyUrl('anthropic'),
  apiKey: 'unused',
  defaultHeaders: ar.getAuthHeaders(),
});
```

### Plain `fetch` (no SDK)

```typescript
const response = await fetch(`${process.env.AGENTROOT_PROXY}/openai/v1/chat/completions`, {
  method: 'POST',
  headers: {
    Authorization: `Bearer ${process.env.AGENTROOT_TOKEN}`,
    'X-AgentRoot-ID': process.env.AGENTROOT_AGENT_ID!,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    model: 'gpt-4o',
    messages: [{ role: 'user', content: 'Hello' }],
  }),
});
```

## Helper methods

```typescript
ar.getProxyUrl('openai');
// → "https://proxy.agentroot.app/openai"

ar.getAuthHeaders();
// → { Authorization: "Bearer art_live_xxx", "X-AgentRoot-ID": "0x7a3f..." }

const health = await ar.healthCheck();
// → { proxy: "ok", token: "valid", vault: "ok", ttlSeconds: 72400 }
```

`getProxyUrl` and `getAuthHeaders` are designed for injection into third-party SDKs that expect a base URL plus headers. `healthCheck` confirms proxy reachability, token validity, and vault status — useful for liveness probes and CI smoke tests.
