# Introduction

AgentRoot is an identity, access control, and API proxy platform for autonomous AI agents. It gives operators a single registration that provides onchain identity, managed API proxy, granular kill switches, and full activity logging.

**One registration. Every API. Full control. Instant kill.**

## The Problem

AI agents make thousands of API calls — to LLMs, data feeds, exchanges, SaaS tools. Today, operators paste API keys into `.env` files that any plugin can read. There's no per-agent visibility, no granular access control, and no way to instantly stop a rogue agent at 3am without SSH-ing into a server.

AgentRoot fixes this by inserting a proxy layer between your agents and the APIs they call. Your keys never touch the agent runtime. Every call is logged. Every API can be toggled on or off. And if something goes wrong, you have a kill switch — from your dashboard, your phone, or your Ethereum wallet.

## Who It's For

**Operators** are people or companies that deploy AI agents. You use the AgentRoot dashboard to register agents, manage API keys, monitor activity, and control access. You don't need to write code — the dashboard handles everything.

**Developers** build agent systems and integrate AgentRoot into their workflows. You use the SDK to authenticate agents, route API calls through the proxy, and handle session tokens. The SDK is a lightweight TypeScript library.

## Core Concepts

### Agent

An agent is a registered AI system that makes API calls through the AgentRoot proxy. Each agent has a unique onchain identity (EAS attestation), a cryptographic keypair (P-256), and a session token for authenticating with the proxy. Agents never hold API keys.

### Operator

The person or company that registered the agent. The operator controls the agent's access, holds the kill switch, and owns the onchain attestation from their Ethereum wallet.

### Proxy

The Cloudflare Worker at `proxy.agentroot.app` that sits between your agent and upstream APIs. It validates the agent's session token, injects the correct API key from Vault, forwards the request, logs everything, and returns the response. Sub-100ms latency at the edge.

### Vault

HashiCorp Vault stores all secrets: API keys (encrypted, versioned), P-256 keypairs (private key never exported), and session tokens. If Vault is unavailable, the system fails closed — no requests are processed.

### EAS Attestation

An immutable onchain record on Base (Coinbase L2) that contains the agent's public key and links it to the operator's wallet. This is the root of trust. It can be verified by anyone, and revoked only by the operator — not even AgentRoot can override it.

### Kill Switch

Four independent mechanisms to stop an agent, from surgical to nuclear. Toggle one API off. Pause the whole agent. Revoke the onchain attestation. Or let auto-kill triggers fire based on usage limits. Details in the [Kill Switch guide](kill-switch).

### Session Token

A short-lived credential (format: `art_live_xxxxxxxxxxxx`) issued by Vault with a 24-hour TTL. This is what the agent uses to authenticate with the proxy. If a token is compromised, it expires automatically in 24 hours. You can also kill it immediately from the dashboard, or revoke the agent's EAS attestation onchain from your wallet — a sovereign kill that works even if AgentRoot is offline. The underlying identity is unaffected.

## How It Works (30-Second Version)

```
1. Operator registers an agent → gets onchain EAS attestation + session token
2. Operator adds API keys → keys are encrypted and stored in Vault
3. Agent makes API calls → sends requests to proxy.agentroot.app
4. Proxy validates token → injects API key → forwards to upstream → logs everything
5. Operator monitors → dashboard shows all activity per-agent, per-API
6. Something goes wrong → kill switch stops the agent instantly
```

The agent's `.env` file contains only three values:

```
AGENTROOT_AGENT_ID=0xABC123...
AGENTROOT_TOKEN=art_live_xxxxxxxxxxxx
AGENTROOT_PROXY=https://proxy.agentroot.app
```

No API keys. No private keys. No secrets that matter if leaked beyond the token's 24-hour TTL.

## What AgentRoot Is Not

AgentRoot is **not** an agent framework. It doesn't help you build agents — use LangChain, CrewAI, AutoGPT, or whatever you prefer. AgentRoot sits underneath, securing the API access layer regardless of which framework you use.

AgentRoot is **not** a payment system. It doesn't handle agent-to-agent payments or micropayments. It's the identity and access control layer that works alongside payment protocols.

AgentRoot is **not** custodial in the Web3 sense. Your onchain attestation lives on Base (Ethereum L2). You control it from your wallet. We manage the Vault and proxy infrastructure, but the root of trust belongs to you.
